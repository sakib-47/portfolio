About

From engineering fundamentals to cloud control frameworks

I'm a final-year engineering student, building a focused path into Cloud Security — starting as a Cloud GRC Analyst and growing toward Cloud Security Engineer. My work sits at the intersection of two things I care about: understanding why a control exists (ISO 27001, NIST CSF, SOC 2, CIS Benchmarks) and being able to build the tooling that checks it automatically, in Go.

Rather than treating security knowledge and software engineering as separate tracks, I'm building both together — every compliance concept I learn gets turned into working code within the same week, tested against real AWS environments on the Free Tier.

A Go CLI that audits live AWS accounts against the CIS AWS Foundations Benchmark: flags IAM wildcard permissions and unused keys, public/unencrypted S3 buckets, and open security groups — then outputs a pass/fail report mapped to control IDs.
